# Roles and Responsibilities

This document outlines the different roles and the Google Accounts required to set up and manage the Google Drive Permission Manager. It's critical to use the correct account for each task.

| Role                      | Account Type                                                  | Relevance                 | Responsibilities                                                                                                                                                                                            |
| :------------------------ | :------------------------------------------------------------ | :------------------------ | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **System Installer & Owner** | Google Workspace account with **Super Admin** privileges.      | **Setup & Core Maintenance** | <ul><li>The primary account used for the entire initial setup process.</li><li>Must be used to authenticate the command-line tools (`gcloud`, `gemini`).</li><li>Creates the control spreadsheet, becoming the owner of the bound Apps Script.</li><li>The script executes with this account's authority.</li></ul> |
| **Sheet Editor**          | Any Google Account.                                           | **Day-to-Day Operation**  | <ul><li>A trusted user who manages folder permissions by editing the control spreadsheet.</li><li>Adds or removes users from the various permission sheets.</li><li>Does not require Super Admin privileges.</li><li>Managed via the `SheetEditors` control sheet.</li></ul> |
| **Managed User**          | Any Google Account.                                           | **End User**              | <ul><li>A regular user who is granted view or edit access to folders managed by the system.</li><li>Their access is determined by their entry in the permission sheets.</li></ul> |
